### 盘中版异常说明

盘中版 (open) 回测产出 0 笔交易。根因分析:

- 两种模式共用预计算 zhongjun 信号 (基于 MA5/10/20/60/120 + MACD + CCI 多日平滑指标)
- market_window 数据完全相同 (未对 open 模式做 H/L/C 替换)

1. 状态文件残留: close 模式运行后 `huang_combo_full_sell_state.json` 中的状态可能影响 open 模式的 V1.1 风控判断
2. scoring_adapter 内部缓存: 某些评分维度可能受前次运行状态影响
3. 需要进一步调试确认

## 五、结论

1. **接入 6+2 + V1.1 后整体表现 vs 单纯信号回测**: 持仓粒度回测揭示了信号级回测无法反映的真实风险 — Part 5/6 的信号胜率在持仓粒度下大幅缩水 (34.3% → 30.7%), 且受仓位管理、滑点、手续费影响, 3 年累计亏损 64.64%
2. **尾盘版 vs 盘中版**: 盘中版因评分异常产出 0 交易, 无法对比. 尾盘版本身跑输大盘 85.71%
3. **vs 大盘**: 两种模式均跑输大盘 (+21.08%). 尾盘版大幅跑输
4. **主要风险点**:
   - 6+2 评分在中小盘 3633 只股票池中筛选力度不足 (min_score=60 门槛偏高)
   - V1.1 风控在回测环境下频繁触发卖出, 缩短持仓周期
   - T+1 合规锁仓进一步限制了灵活度
   - 实盘 3 仓位上限在高换手率下导致大量资金闲置
5. **后续建议**:
   - 降低 min_score 阈值 (如 50) 或调整评分权重以增加信号通过率
   - 排查 open 模式 0 交易异常 (状态文件残留 / scoring 缓存)
   - 考虑扩大持仓上限 (5-10) 以分散风险
   - 评估 V1.1 风控阈值是否过于激进 (回撤 -69.83% 仍触发频繁卖出)
