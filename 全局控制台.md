## 五、操作规则

### 新项目命名

```text
Project_01_项目名
Project_02_项目名
...
```

### 每个项目固定三个工作文档

```text
项目入口或共享任务本.md
项目总览与下一步.md
问题与报错日志.md
```

### 归档原则

1. `Project_*` 是导航/控制层，不替代原始代码、SPEC、agent_hub。
2. 现有工程文件默认链接，不搬家；只有明确要重构路径时才整体移动。
3. 大型回测结果、DuckDB、行情 CSV 不进 `Project_*`，只写路径引用。
4. Obsidian 知识库只放长期沉淀；`knowledge_base` 是 junction，只链接不复制。
5. 超过 10 分钟的任务必须入板；没有交付物路径不算完成；没有验收不进 DONE。

### 状态流转

```text
INBOX → TODO → DOING → WAITING/REVIEW → DONE
                    ↘ BLOCKED
```

任务清单列一下
