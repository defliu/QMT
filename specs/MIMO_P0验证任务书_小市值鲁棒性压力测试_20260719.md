## 三、Phase 1：分行情区间小市值alpha验证（P0，优先级最高）

### 目标
验证小市值alpha在不同市场环境下的稳定性，确认是真因子还是只是牛市β。

### 测试组合（固定最优参数，与路径A TOP1完全一致）
```
市值区间：0-30亿，调仓频率：双月，持仓数：TOP80，止损线：-12%
定价方式：Open定价，动态universe：开启
```

### 区间划分（5个区间，全部与2018.07.31基准对齐）
| 区间 | 起止日期 | 性质 | 关键事件 |
|------|----------|------|----------|
| 1 | 2018.07.31 - 2018.12.31 | 熊市后半段 | 贸易战最剧烈阶段，全年普跌 |
| 2 | 2019.01.01 - 2021.12.31 | 牛市 | 核心资产行情，小市值牛市起点 |
| 3 | 2022.01.01 - 2023.12.31 | 震荡市 | 疫情后经济弱复苏，区间震荡 |
| 4 | 2024.01.01 - 2024.06.30 | 极端压力测试 | 微盘股崩盘，量化踩踏 |
| 5 | 2024.07.01 - 2026.06.18 | 结构性牛市 | 政策底后修复，小市值反弹 |

### 对照组（每个区间同时跑3组）
- 组A：0-30亿（核心组）
- 组B：全池（对照组1）
- 组C：80亿+（对照组2）

### 🔒 分区间前瞻保护规则（严格遵守）
1. 每个区间**独立运行**，区间起始点的所有因子数据只用起始点之前的数据
2. 区间内调仓逻辑、universe更新逻辑与全周期完全一致
3. 禁止用区间结束后的数据计算任何指标（包括流动性指标、因子数据）
4. 所有区间的起始universe都用该区间起始日的真实可获得数据
5. 此区间划分基于事后知识，仅用于压力测试，不代表实盘可预知

### 输出指标（每组区间×3组）
1. 年化收益
2. 夏普比率
3. 最大回撤
4. 胜率
5. 止损次数
6. 调仓次数
7. 区间累计收益曲线（月度数据点，用于画图）

### 通过标准（相对判定，不是绝对阈值）
- ✅ 每个区间0-30亿 vs 80亿+的**超额收益 > 3%** → 小市值是真alpha
- ⚠️ 超额收益 1-3% → 有alpha但不稳定
- ❌ 超额收益 < 1% → 小市值是风格β，策略不可长期依赖
- ❌ 2024微盘崩盘区间最大回撤 > -30% → 极端风险过高，实盘不可行

### 预期工作量
5个区间 × 3组 = 15组回测 ≈ 4小时

## 四、Phase 2：流动性约束回测（P0，同等优先级）

### 目标
量化实盘冲击成本和流动性约束对收益的衰减，回答"回测15%，实盘能拿到多少"。

### 基准组合
与Phase 0完全一致：0-30亿 + 双月 + TOP80 + 止损-12%，全周期回测

### 🔒 流动性过滤执行顺序（严格遵守）
1. 先按市值过滤：0-30亿
2. 再按成交额过滤：调仓日T日的**前20个交易日**（T-20到T-1）日均成交额 > 阈值
3. 最后打分选TOP N
4. 股票不足处理：如果过滤后股票数 < TOP N，有多少选多少，不强制凑数

### 测试方案A：成交额阈值过滤（用filter_func实现）
在0-30亿池子基础上，追加成交额过滤：
- 阈值1：2000万（宽松，适合<500万资金）
- 阈值2：3000万（中等，适合500万-2000万资金）
- 阈值3：5000万（严格，适合>2000万资金）

**实现方式**：在`scoring.py`的`filter_func`中追加流动性检查逻辑，数据从E盘日线parquet的`amount`列获取（如无此字段则先从astock补充）。

### 测试方案B：分级交易成本模型（直接改backtest参数）
当前回测用固定0.2%单边费率。用更贴近实盘的分级成本：
- 小资金（<1000万）：0.15%单边（佣金+印花税+低滑点）
- 中资金（1000万-5000万）：0.3%单边（含中等滑点）
- 大资金（>5000万）：0.5%单边（含高滑点+冲击成本）

**注意**：方案B（单票仓位上限1%/2%/3%）本次暂不执行——当前`backtest.py`不支持`max_weight_per_stock`参数，需后续专门适配。

### 输出指标（每个测试组）
1. 年化收益（vs基准15.5%的衰减幅度）
2. 夏普比率
3. 最大回撤
4. 过滤后剩余股票数（方案A）
5. 策略容量估算（基于流动性+换手率）

### 通过标准
- ✅ 严格约束（成交额>5000万 + 0.5%成本）后年化仍 > 10% → 回测真实可信
- ⚠️ 严格约束后年化 7-10% → 有衰减但仍可接受
- ❌ 严格约束后年化 < 7% → 回测虚高，需调整持仓数/市值区间

### 预期工作量
方案A 3组 + 方案B 3组 = 6组回测 ≈ 3小时

## 五、错误弹性与执行规则

### 单组回测失败处理
1. 记录完整ERROR栈到`p0_validation.log`
2. 跳过该组，继续执行下一组
3. 最终报告中标注失败组及原因
4. 如果失败组数 > 4组（22组中>18%），整体任务标记为失败

### 执行顺序
Phase 0（预检）→ Phase 1（分区间）→ Phase 2（流动性）

### 禁止事项
❌ **禁止改动打分逻辑/因子/参数**：本任务是纯验证，不是优化，所有基准参数与路径A TOP1完全一致
❌ **禁止跳过2024微盘崩盘区间**：这是小市值策略的终极压力测试，必须单独跑
❌ **禁止用全周期数据混跑**：每个区间必须独立截断，独立回测，不能用未来数据
❌ **禁止在Phase 0不通过的情况下继续跑后续阶段**
