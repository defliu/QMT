# Task: Add Limit-Down Detection + Deferred Sell Queue to strategy_main.py

## Problem
When a stock hits limit-down, the sell order stays unfilled. The current code retries indefinitely (no max retry for sells, no `_is_limit_down()` function). The stock needs to be sold when limit-down lifts.

## What to Add

### 1. `_is_limit_down(code, C)` function
Mirror of existing `_is_limit_up()`. Check if the stock's current price has dropped to the exchange's limit-down threshold:
- Main board (6xxxxx, 0xxxxx, 3xxxxx not 300): `pct <= -9.5`
- ChiNext (300xxx, 301xxx) / STAR (688xxx, 689xxx): `pct <= -19.5`
- Beijing (.BJ): `pct <= -29.5`

Use the same two-tier fallback (get_full_tick first, then df close comparison).

### 2. `_g_pending_limitdown_sells` global dict
A new queue for stocks that failed to sell due to limit-down:
```python
_g_pending_limitdown_sells = {}  # {code: {info dict}}
```
Each entry stores: code, original order_id, volume, sell_price, cost, timestamp.

### 3. Modify `_check_pending_sells()`
When a sell order is unfilled and `_is_limit_down(code, C)` is True:
- Cancel the order
- Move the stock from `_g_pending_sells` to `_g_pending_limitdown_sells`
- Print log: `[跌停暂缓] %s 跌停中，移入等待队列`

### 4. New function: `_check_limitdown_sells(C, today)`
Called each trading cycle (before or after `_check_pending_sells`). For each stock in `_g_pending_limitdown_sells`:
- **If `_is_limit_down()` is False** (limit-down lifted): execute sell at current price, move back to `_g_pending_sells`
- **If still limit-down**: keep waiting
- **Time limit**: if waiting > 5 trading days, force sell at market price (aggressive fallback) and log warning

### 5. Modify `_execute_trade()` (the main trading function)
Add call to `_check_limitdown_sells()` so it runs every cycle.

### 6. Max retry limit for sells
In `_check_pending_sells()`, for unfilled non-limit-down sells:
- Track `retries` count (already done in `_retry_pending_sell`)
- If `retries >= 3`: cancel order, log error, remove from pending. Do NOT retry indefinitely.
- Do NOT apply this limit to limit-down detected stocks (those go to the special queue instead).

### Logic flow (sell path):

```
Sell triggered → _g_pending_sells[code]
  → each cycle: _check_pending_sells()
    → order found, filled → done
    → order found, unfilled:
      → _is_limit_down(code, C)?
        → YES → cancel order → move to _g_pending_limitdown_sells
        → NO → _retry_pending_sell (max 3 times)
          → 3+ retries → cancel, log error, remove
    → order not found, position zero → done
    → order not found, position >0 → retry

  → each cycle: _check_limitdown_sells()
    → for each in _g_pending_limitdown_sells:
      → _is_limit_down() still true?
        → YES → check age > 5 days?
          → >5: force sell at market (use current price, accept discount)
          → ≤5: keep waiting, do nothing
        → NO (limit-down lifted):
          → execute sell at current price → move back to _g_pending_sells
```
