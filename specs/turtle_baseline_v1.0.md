# SPEC: 海龟策略A股基线回测 v1.0

## Objective

验证海龟交易法原版参数在A股个股上的基线表现。不改逻辑、不加过滤、不做优化，纯粹用数据回答一个问题：**原版海龟规则在A股能不能赚钱？**

回测脚本独立运行（Python 3.10），不进QMT build，不依赖现有策略体系。

## 策略规则（原版海龟，A股适配最小改动）

### 1. 入场信号

```
条件: 当日收盘价 > 过去20个交易日（不含当日）的最高收盘价
动作: 次日开盘价买入
```

- 用收盘价突破，不用最高价突破（A股收盘价更稳定，减少盘中假突破）
- 次日执行，不当日执行（T+1约束）

### 2. 仓位计算

```
ATR = 过去20个交易日的ATR值（True Range的平均）
单笔风险 = 账户初始资金 × 1%
买入股数 = 单笔风险 / (2 × ATR / 当前价 × 100)
```

- 账户初始资金：1,000,000元（100万）
- 每只股票独立计算仓位，不考虑组合层面的资金分配
- 单票仓位上限：账户资金的20%（硬限制，防止ATR极小时满仓一只）

### 3. 止损

```
初始止损价 = 买入价 - 2 × ATR(20)
每日更新ATR
移动止损：持仓期间，止损价 = max(初始止损价, 当日最高价 - 2 × ATR)
触发：当日收盘价 < 止损价 → 次日开盘卖出
```

### 4. 离场（趋势退出）

```
条件: 当日收盘价 < 过去10个交易日（不含当日）的最低收盘价
动作: 次日开盘价卖出
```

- 止损和离场独立判断，哪个先触发就执行哪个
- 执行后不再开仓，等待下一次入场信号

### 5. 不做的事（明确排除）

| 排除项 | 理由 |
|--------|------|
| ❌ 加仓 | 基线阶段只测单笔入场+单笔离场，排除加仓干扰 |
| ❌ 做空 | A股做空不便利，基线只做多 |
| ❌ 量能过滤 | 基线不加任何过滤条件 |
| ❌ 大盘环境过滤 | 基线不看大盘，纯个股信号 |
| ❌ 板块过滤 | 基线不做行业/板块筛选 |
| ❌ 滑点优化 | 固定0.1%滑点，不做动态滑点 |

## Code Style

- 独立Python脚本，不依赖QMT build系统
- Python 3.10运行（与现有回测脚本一致）
- 使用 pandas + mootdx 取数
- 不使用 dict[str,...] / str | None / match-case（QMT 3.6.8兼容习惯，虽然本脚本不进QMT，但保持一致风格）
- 数据缓存到本地parquet，避免重复下载
- 脚本必须有 `if __name__ == "__main__"` 入口
- 输出目录自动创建

## Testing

回测脚本运行后的验证步骤：

1. **数据完整性**：检查300只股票的数据是否有缺失日期，缺失日期比例<1%视为合格
2. **交易合理性**：随机抽3只股票，人工核对入场/离场信号是否符合规则
3. **成本正确性**：随机抽1笔交易，手工计算含手续费+滑点的盈亏，与脚本输出比对
4. **边界情况**：检查是否存在持仓期ATR=0导致除零的情况
5. **输出完整性**：4个输出文件全部生成且非空
