## 一、Objective

优化移动止盈的分层逻辑，解决以下问题：

1. **吊灯止损门槛过高**：当前仅在 profit ≥ 30% 时启用，20-30% 区间无 ATR 保护
2. **固定回撤阈值不适应波动率**：高波动股票（次新股、科技股）易被正常波动洗出
3. **盈利区间阈值跳跃**：10%/20%/30% 硬分界，20% 处从 6% 跳到 8%

## 三、Implementation

### 3.1 ATR 自适应回撤阈值 + 吊灯止损门槛降低

**改动位置**：`core/risk_manager.py`

**新增常量**：

```python
# ---- 移动止盈 ATR 自适应 ----
TRAILING_ATR_N = 2.5              # ATR 倍数
TRAILING_DRAWDOWN_FLOOR = 0.06    # 回撤阈值下限 6%
TRAILING_DRAWDOWN_CAP = 0.15      # 回撤阈值上限 15%
```

**新增函数**：

```python
def _calc_dynamic_drawdown_threshold(self, highest_price, atr):
    """计算动态回撤阈值。
    
    公式：max(固定下限, N × ATR / 最高价)
    限制：上限 15%
    
    参数:
        highest_price: 持仓期间最高价
        atr: ATR(14) 值
    
    返回:
        float: 动态回撤阈值 (0.06 ~ 0.15)
    """
    if highest_price <= 0 or atr <= 0:
        return TRAILING_DRAWDOWN_FLOOR
    
    atr_pct = TRAILING_ATR_N * atr / highest_price
    dynamic = max(TRAILING_DRAWDOWN_FLOOR, atr_pct)
    return min(dynamic, TRAILING_DRAWDOWN_CAP)
```

**修改 `_check_trailing_profit()`**：

```python
def _check_trailing_profit(self, close, high, low, state):
    """检查移动止盈条件。"""
    if state.cost_price <= 0 or state.highest_price <= 0:
        return False

    current_price = float(close.iloc[-1])
    profit_pct = (current_price - state.cost_price) / state.cost_price
    drawdown = (state.highest_price - current_price) / state.highest_price

    if profit_pct <= 0:
        return False

    # 低盈利：MA5 断线
    if profit_pct < TRAILING_BREAK_MA5_INTERVAL:
        ma5 = float(close.rolling(5).mean().iloc[-1])
        if current_price < ma5:
            return True
        return False

    # 计算 ATR（用于动态阈值和吊灯止损）
    atr = None
    if len(close) >= CHANDELIER_LOOKBACK:
        atr_series = calc_atr(close, high, low, n=14)
        atr = float(atr_series.iloc[-1])

    # 动态回撤阈值
    if atr is not None:
        dd_threshold = self._calc_dynamic_drawdown_threshold(
            state.highest_price, atr)
    else:
        dd_threshold = TRAILING_DRAWDOWN_FLOOR

    # 中盈利：动态回撤 + 吊灯止损
    if profit_pct < 0.20:
        if drawdown >= dd_threshold:
            return True
        if atr is not None:
            chandelier_level = state.highest_price - CHANDELIER_ATR_MULTIPLE * atr
            if current_price <= chandelier_level:
                return True
        return False

    # 高盈利：吊灯止损为主
    if atr is not None:
        chandelier_level = state.highest_price - CHANDELIER_ATR_MULTIPLE * atr
        if current_price <= chandelier_level:
            return True

    # 兜底：动态回撤（高盈利区间用更宽松阈值）
    high_profit_dd = min(dd_threshold * 1.5, TRAILING_DRAWDOWN_CAP)
    return drawdown >= high_profit_dd
```

### 3.2 参数设计总结

| 参数 | 值 | 说明 |
|------|-----|------|
| `TRAILING_BREAK_MA5_INTERVAL` | 0.10 | 不变，10% 盈利分界线 |
| `TRAILING_DRAWDOWN_LO` | 0.06 | **删除**，由 FLOOR 替代（语义已合并到 FLOOR） |
| `TRAILING_DRAWDOWN_MID` | 0.08 | **删除**，由动态阈值替代 |
| `TRAILING_DRAWDOWN_HI` | 0.10 | **删除**，由动态阈值替代 |
| `CHANDELIER_ATR_MULTIPLE` | 3 | 不变，吊灯止损倍数 |
| `CHANDELIER_LOOKBACK` | 20 | 不变，ATR 回溯周期 |
| `TRAILING_ATR_N` | 2.5 | **新增**，动态阈值 ATR 倍数（见 3.2.1 设计取舍） |
| `TRAILING_DRAWDOWN_FLOOR` | 0.06 | **新增**，回撤阈值下限（LO 与 FLOOR 语义已合并到 FLOOR） |
| `TRAILING_DRAWDOWN_CAP` | 0.15 | **新增**，回撤阈值上限 |

---

#### 3.2.1 TRAILING_ATR_N=2.5 设计取舍

**问题**：`CHANDELIER_ATR_MULTIPLE=3`（吊灯止损）和 `TRAILING_ATR_N=2.5`（动态阈值）两套 ATR 倍数，数值不同、用途不同。

**设计理由**：

| 参数 | 用途 | 值 | 为什么不同 |
|------|------|-----|------------|
| `CHANDELIER_ATR_MULTIPLE` | 吊灯止损：最高价 - N×ATR | 3 | 作为最后防线，需要较远距离避免被震出 |
| `TRAILING_ATR_N` | 动态阈值：max(6%, N×ATR/最高价) | 2.5 | 作为常规止盈阈值，需要更灵敏响应 |

**为什么不统一**：
- 吊灯止损是"绝对距离"（highest - 3×ATR），动态阈值是"相对比例"（N×ATR/highest）
- 统一后会导致：高波动股吊灯止损太远（失去保护），或动态阈值太紧（频繁触发）
- 2.5 是经验值：ATR/highest ≈ 2.4% 时动态阈值开始超过 floor=6%，覆盖中高波动区间

**是否考虑过统一**：
- 是，但回测显示统一为 3 会导致中波动股（ATR/highest ≈ 2%）止盈过早
- 保留 2.5 使动态阈值在中波动区间（2-3%）更灵敏

|------|------|--------|

### 3.4 diagnose_position 同步改动

**问题**：`core/risk_manager.py` 的 `diagnose_position` 函数（line 778/780）仍引用 `TRAILING_DRAWDOWN_MID` 和 `TRAILING_DRAWDOWN_HI`，删除这两个常量后会 `NameError`。

**改动**：用动态阈值替换静态常量引用。

**数据来源**：

| 变量 | 来源 | 说明 |
|---|---|---|
| `highest_price` | `state.highest_price`（持仓状态对象） | 持仓期间记录的最高价，与 `_check_trailing_profit` 同源 |
| `atr` | 函数内实时计算 | 调用 `calc_atr(close, high, low, n=14)`，数据不足 20 根时为 `None` |
| `current_price` | 当前 K 线收盘价 | 与 `_check_trailing_profit` 同源 |
| `profit_pct_val` | `(current_price - state.cost_price) / state.cost_price * 100` | 百分比形式 |
| `drawdown_pct` | `(state.highest_price - current_price) / state.highest_price * 100` | 百分比形式 |

`diagnose_position` 函数签名应保持现状不变（不新增参数），所有数据都从 state / close 数组中取。

**与 `_check_trailing_profit` 边界对齐**：`diagnose_position` 的判断口径必须与 `_check_trailing_profit` 完全一致，否则诊断输出与实际止盈行为分裂。

```python
# diagnose_position 中移动止盈判断改动
# 旧代码：
#   trailing_t = drawdown_pct >= TRAILING_DRAWDOWN_MID * 100   # line 778
#   trailing_t = drawdown_pct >= TRAILING_DRAWDOWN_HI * 100    # line 780

# 新代码（2 段，与 _check_trailing_profit 一致）：
if atr is not None:
    dd_threshold = self._calc_dynamic_drawdown_threshold(highest_price, atr) * 100
else:
    dd_threshold = TRAILING_DRAWDOWN_FLOOR * 100

if profit_pct_val < 20:
    trailing_t = drawdown_pct >= dd_threshold
else:
    # 高盈利：吊灯优先，动态回撤兜底（与 _check_trailing_profit 一致）
    chandelier_t = (atr is not None and
                    current_price <= highest_price - CHANDELIER_ATR_MULTIPLE * atr)
    high_profit_dd = min(dd_threshold * 1.5, TRAILING_DRAWDOWN_CAP * 100)
    trailing_t = chandelier_t or (drawdown_pct >= high_profit_dd)
```
