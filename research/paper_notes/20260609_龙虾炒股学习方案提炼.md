## 一、核心结论

这套资料的核心不是让 AI 学会“预测股票必涨”，也不是学“主力/庄家玄学”。

真正有价值的是一个短线交易决策框架：

```text
先看市场环境
  ↓
再看主线方向
  ↓
再看个股地位
  ↓
再看买点/节奏/确认
  ↓
再看风险收益比
  ↓
最后给出明确交易结论
```

适合转化为咱们量化研究中的：
- 环境过滤器
- 板块/方向强度模块
- 个股评分模块
- 买点确认模块
- 风险排除模块
- 仓位建议模块
- 策略输出枚举

---

## 二、不要学习的东西

资料里反复强调：不要把交易分析搞成玄学。

应避免：

1. 主力洗盘口诀
2. 庄家暗语
3. 一眼看穿主力意图
4. 主力一定会怎样
5. 看到大单就过度解读
6. 看到涨停就默认值得追
7. 情绪化追高
8. 因为错过而乱追
9. 因为亏损而报复性交易

转化到咱们流程里：

> 模型可以提出假设，但不能替代回测。任何“资金意图/主力行为”的说法，都必须落成可观测指标，否则不进入策略。

## 三、机构思维模块

机构思维的重点是：逻辑、预期差、估值、景气度、风格、风险收益比。

### 1. 预期差思维

核心问题：
- 当前市场预期是什么？
- 实际情况比预期更好还是更差？
- 是不是超预期驱动？
- 是否已经被提前交易？

量化转化：
- 公告/业绩/政策事件后相对强度
- 事件后成交量变化
- 事件前涨幅是否已透支
- 财报超预期后的持续性

### 2. 估值思维

核心问题：
- 当前估值高不高？
- 好消息是否已经反映？
- 现在买是否还有性价比？

量化转化：
- PE/PB/PS 分位数
- 行业内估值分位
- 估值变化与价格涨幅背离
- 高估值 + 高涨幅 降权

### 3. 景气度思维

核心问题：
- 行业景气度上行、持平还是下行？
- 是短期刺激还是中期改善？
- 方向还有没有持续性？

量化转化：
- 行业指数相对强度
- 行业内上涨家数占比
- 行业成交额占比提升
- 行业盈利/营收增速趋势

### 4. 资金风格思维

核心问题：
- 当前市场偏大票还是小票？
- 偏成长还是红利？
- 偏进攻还是防御？
- 目标票顺风格还是逆风格？

量化转化：
- 大小盘风格强弱
- 成长/价值风格强弱
- 高股息/题材/科技等风格轮动
- 市场成交额与风险偏好

### 5. 风险收益比思维

核心问题：
- 方向对，不等于现在值得买。
- 如果做错，亏多少？
- 如果做对，空间多少？
- 这笔交易值不值得？

量化转化：
- 上方空间/下方止损比
- 近期波动率
- 距离压力位/支撑位
- 涨幅透支度
- 高位加速降权

## 四、交易员思维模块

交易员思维的重点是：节奏、买点、确认、试错、撤退。

### 1. 节奏思维

方向判断对，不代表现在就该买。

检查问题：
- 是早了、晚了，还是刚好？
- 当前是追高风险大，还是回踩机会更好？
- 是先手窗口还是后手接力？

量化转化：
- 当日涨幅位置
- 近 N 日涨幅
- 距离均线/突破位偏离
- 回踩不破/突破确认

### 2. 买点思维

不能只说“看好”，必须明确买点。

买点类型：
- 突破买
- 回踩买
- 企稳买
- 分时转强买
- 板块共振确认买

量化转化：
- 突破前高
- 回踩均线不破
- 放量确认
- 分时站上均价线
- 板块同步走强

### 3. 确认思维

看起来强，不等于已经确认强。

确认项：
- 量价确认
- 板块确认
- 关键位置确认
- 分时强度确认

量化转化：
- 放量突破
- 板块涨幅/上涨家数
- 个股相对板块强度
- 关键价位站稳

### 4. 试错思维

不是所有交易都要等 100% 确认，但试错必须轻仓、有条件、有退出标准。

量化转化：
- 环境 B 级以下只允许试错仓
- 条件不完整时降低目标仓位
- 试错失败触发快速退出

### 5. 撤退思维

买之前必须定义错判条件。

检查问题：
- 什么信号说明判断错了？
- 什么价格/形态/板块变化必须退出？
- 买错后怎么处理？

量化转化：
- 止损线
- 跌破关键均线
- 板块强度下降
- 个股评分跌破阈值
- 次日不及预期退出

## 五、游资短线思维模块

适用于短线强势股、热点股、题材股。

### 7 个核心维度

1. 情绪周期：启动 / 发酵 / 高潮 / 分歧 / 退潮
2. 辨识度：是否题材龙头、核心强势股、市场记忆点
3. 板块联动：是否单兵作战，还是板块共振
4. 承接：冲高后有没有资金接住
5. 节奏窗口：先手、后手、太晚、合理窗口
6. 高位风险：最强也可能最危险
7. 次日预期：明天是否还有继续认可的空间

### 短线对象分类

- 情绪龙头
- 逻辑核心
- 板块跟风
- 高位危险股
- 可低吸观察对象
- 不值得做的噪音对象

### 关键原则

> “强”不等于“值得追”。

> “今天最强”可能意味着“明天最危险”。

量化转化：
- 龙头/前排地位评分
- 板块共振评分
- 高位加速风险惩罚
- 次日溢价预期因子
- 跟风补涨降权

## 六、盘口资金思维模块

盘口模块的价值在于辅助买点，不是玄学判断主力。

### 7 个核心维度

1. 分时强弱：是否沿均价线上方强运行
2. 承接强弱：回落是否有人接
3. 冲高回落识别：冲高后站稳还是失守
4. 主动性资金：持续主动买入还是一波脉冲
5. 压力与抛压：上方是否一冲就被砸
6. 真假强势：真强还是假强
7. 盘口与 K 线结合：盘口不能脱离日线位置和板块

### 盘口状态分类

- 强承接真强盘口
- 冲高回落承压盘口
- 假突破盘口
- 弱分时震荡盘口
- 高位危险盘口
- 可继续盯的强势盘口
- 不值得追的诱多盘口

量化转化：
- 分时均价线上方时间占比
- 冲高回落幅度
- 二次拉起能力
- 主动成交金额变化
- 上影线/放量滞涨
- 高位爆量回落惩罚

## 七、固定决策流程

资料中最值得吸收的是固定工作流：

```text
第一步：市场环境总判断
第二步：方向优先级判断
第三步：个股逐只分析
第四步：个股分级
第五步：优先级排序
第六步：最终建议
```

### 市场环境总评

必须先判断：
- 指数环境：强 / 中 / 弱
- 情绪环境：强 / 中 / 弱
- 赚钱效应
- 主线方向
- 量能是否支持
- 高位股反馈
- 今日适合度
- 环境级别
- 仓位建议

### 环境级别

| 等级 | 含义 | 仓位建议 |
|---|---|---|
| S级 | 可积极出手 | 可适当积极 |
| A级 | 可以参与，但需筛选 | 控制仓位参与 |
| B级 | 轻仓试错 | 轻仓试错 |
| C级 | 观察为主 | 少动/观察 |
| D级 | 原则上不主动出手 | 尽量空仓 |

## 八、固定交易结论枚举

所有标的最终只能归入四类之一：

1. **可直接试仓买入**
   - 逻辑清晰
   - 环境支持
   - 位置/强度/风险收益比合适
   - 当前具备可执行性

2. **条件满足才可买入**
   - 有逻辑，但买点不是最佳
   - 必须等触发条件
   - 例如回踩不破、放量突破、分时转强、板块共振

3. **只可观察，不可买**
   - 可以继续跟踪
   - 但当前买点/位置/风险收益比不支持出手

4. **明确不买**
   - 不符合交易标准
   - 风险明显大于收益
   - 即使热度高也不碰

这套枚举很适合转化为 QMT 策略中的 `decision_state`。

## 九、个股分析结构

每只股票至少要输出：

```text
股票名称+代码：
1. 方向归属：主线 / 次主线 / 跟风
2. 逻辑强度：强 / 中 / 弱
3. 个股地位：核心 / 前排 / 跟风 / 补涨 / 边缘
4. 位置判断：低位启动 / 中位强化 / 高位博弈 / 明显透支
5. 买点判断：好 / 一般 / 差
6. 风险点：2~4条具体风险
7. 操作结论：四类枚举之一
8. 结论理由：2~4句
9. 条件触发：如果是条件买入，写清触发条件
```

如果用于量化，要把这些转化为字段：

```text
market_score
sector_score
stock_role_score
position_score
entry_timing_score
risk_reward_score
liquidity_score
decision_state
position_suggestion
invalid_condition
```

## 十、排除逻辑

资料强调：“先排除，再推荐”。

应显著降级或排除：

1. 高位明显加速后再接
2. 缩量硬顶，承接不明
3. 高开过多，盈亏比差
4. 逻辑不清，只是情绪冲动
5. 题材发散过头，核心不明确
6. 个股不是前排，只是跟风补涨
7. 板块持续性存疑
8. 市场整体环境不支持
9. 冲高回落历史明显
10. 明明强，但不适合当前模式

核心原则：

> 强不等于适合做。

> 涨得好看不等于值得买。

## 十一、和咱们现有工作流的结合

这套资料适合补强 Hermes 的研究与策略讨论能力，但不能直接变成实盘策略。

正确用法：

1. **作为研究框架**
   - 用来和诚哥讨论策略想法
   - 用来拆环境、方向、个股、买点、风险

2. **作为因子设计来源**
   - 把“辨识度、承接、节奏、预期差”等语言转为可计算指标

3. **作为反证清单**
   - 每个策略都问：什么时候不能做？什么情况下必须降级？

4. **作为输出格式约束**
   - 不再模糊说“可以关注”
   - 必须给出分级、排序、条件、风险、是否值得回测

5. **不作为最终交易依据**
   - 最终仍看回测和模拟/实盘表现

## 十二、后续应用规则

以后当诚哥让我分析股票、板块、策略时，默认加入以下检查：

```text
1. 先判断市场环境和仓位，不直接推荐个股。
2. 先排除明显不该做的，再筛选机会。
3. 分清“能看”和“能做”。
4. 每个结论必须明确归类：试仓 / 条件买 / 观察 / 不买。
5. 每个策略必须说明适用行情、失效场景、错判条件。
6. 所有主观概念都要尽量转化为可回测指标。
7. 回测结果是最终验证，不靠模型口头判断。
```
