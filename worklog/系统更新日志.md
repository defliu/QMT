## V1.0 — 流程重构：去 check_buy + 通达信公式 scan（2026-05-31）

### 核心理念修正
**之前错误的假设：** check_buy（买点1/买点2/双色带/MACD）是买入决策的核心。
**实际流程：** 通达信选股（7 条件）→ 池 → 8D 评分 → 买入。check_buy 是冗余门。

### 变更清单

#### 1. bypass check_buy
- `scripts/run_backtest.py:_patch_qmt_wrapper()` — 回测模式 check_buy 替换为始终返回 True
- 池中所有股票直通 8D 评分，不再被 check_buy 卡掉 ~77%

#### 2. 板块热度 mock
- `_compute_mock_sector_heat()` — 每 bar 基于池内近 5 日涨幅排序赋分
- top20% → 5-10 分 | middle40% → 1-4 分 | bottom40% → 0 分
- 注入 `_g_scorer.update_sector_bonus()`
- handlebar 内 `_run_sector_analysis` 返回空 dict，不覆盖已设热度（兼容性验证通过）

#### 3. scan_market 升级为通达信实战放宽版
- 删除原有轻量过滤（价格/ST/上市天数）
- 新增 `_tdx_formula_filter()` — 7 条件全量筛选：

| # | 条件 | 通达信原公式 | Python 等价 |
|---|------|------------|------------|
| 1 | 筹码密集 | COST(95)-COST(5) ≤ 25% | 60 日振幅 ≤ 25% |
| 2 | 突破密集顶 | C > REF(HHV(H,60),1) 三日内突破 | 同逻辑 |
| 3 | 蓄势 | 5 日 ≥ 2 天涨跌幅 < 3% | 同逻辑 |
| 4 | 多头排列 | MA5>10>20 + MA60 向上/走平 | 同逻辑 |
| 5 | MA5 角度 | ≥ 30° | 同逻辑 |
| 6 | 阳线 | C > O | 同逻辑 |
| 7 | 排除急拉坑 | 18 日坑幅 ≥ 16% + 3 日急拉 ≥ 13% | 同逻辑 |

#### 4. 代码清理
- `core/signal_main_rise.py`: 删除 check_buy / _filter_pit_breakout / 双色带 / N,N1,N2,N3,M,YANG_RATIO,CMF_PERIOD,CMF_THRESHOLD（-122 行）
- `adapters/qmt_wrapper.py`: 删除 `from signal_main_rise import check_buy, N, N1, N2, N3, M`
- `tests/test_signal.py`: 删除 TestCheckBuy
- qmt37_strategy/ 保留未删

### 架构图（V1.0）
```
通达信选股（7 条件）
     ↓ 池文件
8D 评分（ScoreCalculator8D + 板块热度 mock）
     ↓ 前 3
买入

--scan 模式: 内置 _tdx_formula_filter() 代替通达信
             7 条件 + mootdx 日 K 线 → 候选池 → 8D 评分
```

### 关键发现
- 通达信实战放宽版选股公式（COST 筹码密集/60 日突破/30°角）与千问版 check_buy_sell（90 日振幅替代/30 日+60 日混合/45°角/MACD/买点分型）**不是同一回事**
- 千问版参数更严、条件更多，在真实交易流中从未实际否决过通达信选出的票
- check_buy 的买点分型、双色带、MACD 条件是另一套验证逻辑，对通达信池子冗余
