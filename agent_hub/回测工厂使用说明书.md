## §2 工厂能做什么 / 不做什么

### ✅ 能

| 能力 | 说明 |
|---|---|
| 离线日线回测 | 从 DuckDB 读 OHLCV，按 universe × 日期循环；next_open / close 两种成交模型 |
| 6+2 评分驱动选股 | 默认调用 `strategy_core.scoring_adapter.score_universe`（核心 6 维 + 情绪 + 板块）|
| PIT universe | 按月切片的候选池，避免幸存者偏差（D 阶段已证伤害，但管线本身支持）|
| benchmark 对照 | 通过 `benchmark_index.duckdb` 加载指数曲线（默认 null，可显式开启）|
| 工厂级 batch grid | yaml 声明 grid 维度，cartesian 展开自动跑 |
| 复现指纹 | 三个 hash + data_mtime + dedup_count + WAL 检测，配置/数据变化自动失效 |
| 短样本期警告 | report / summary / logs / batch summary 全链路标注，防止误用为策略结论 |

### ❌ 不做（v0.2 边界）

| 不做项 | 原因 / 替代方案 |
|---|---|
| 实盘 / 模拟盘下单 | 工厂不 import xtquant / passorder / xttrader，只产 review/probe/results md 文件 |
| 盘中信号 | v0.2 仅日线、收盘后决策、T+1 / T close 成交 |
| 自动补数 | 缺数据请走金策智算 history_sync GUI 或 v0.3 的 `sync_xtquant_to_duckdb.py` |
| 写金策智算 DuckDB | `quantifydata.duckdb` 永久只读；reader 强制 `read_only=True` |
| 写 D 盘大产物 | 决策 J：D 盘只放代码/配置/SPEC，results/cache/sample_db 必须落 F 盘 |
| 写 C 盘任何 backtest 产物 | 决策 J：C 盘完全禁止；tempfile 已重定向到 F 盘 |
| 计算依赖 benchmark 的指标 | `benchmark_available=false` 时超额收益 / 信息比率 / 跟踪误差均输出 null |

## §3 调用入口

### 3.1 单次回测：`backtest.scripts.run_backtest`

```bash
py -3.10 -m backtest.scripts.run_backtest --config <yaml 路径>
```

参数：

| 参数 | 必填 | 说明 |
|---|---|---|
| `--config` | ✅ | yaml 配置文件路径，相对仓库根或绝对均可 |

行为：
1. `init_workspace.ensure_workspace()` — 首次运行自动创建 `F:/backtest_workspace/{results,batch_summary,sample_db,cache,logs,results_archive}` + README.txt。
2. 加载 yaml，计算 `config_hash` / `universe_hash`。
3. 实例化 `DuckDBDailyReader(db_path, data_source)`。
4. `run_backtest(...)` 执行核心循环。
5. `report.write_all(...)` 写 6 文件到 `F:/backtest_workspace/results/<run_id>_<config_name>/`。
6. stdout 打印结果目录路径。

### 3.2 grid 批量：`backtest.scripts.run_batch`

```bash
py -3.10 -m backtest.scripts.run_batch --experiment <yaml 路径>
```

参数：

| 参数 | 必填 | 说明 |
|---|---|---|
| `--experiment` | ✅ | grid 配置 yaml（含 batch / grid 两段） |

行为：
1. 读取 `batch.base` 指向的基准 yaml。
2. `grid.<dotted-path>` 笛卡尔展开（如 `strategy.max_positions: [3,5,8]` × `strategy.min_score: [55,60]` → 6 个 leaf）。
3. 每个 leaf 串行调用单回测。
4. 汇总 24 列 CSV → `F:/backtest_workspace/batch_summary/<batch_id>_<timestamp>.csv`。

> 重要：`run_batch` **不**加载 `clean_results.py`（硬约束 #6）；retention 由人工 / cron 触发。

### 3.3 辅助脚本（按需调用）

| 脚本 | 用途 | 是否进 import 链 |
|---|---|---|
| `init_workspace.py` | 首次创建 F 盘 workspace 目录树 | runner 自动调（幂等） |
| `clean_results.py` | results 30 天移 archive、90 天才允许真删；默认 dry-run | **独立 CLI，绝不进 reader/engine** |
| `validate_data.py` | 校验 DuckDB 覆盖度 / dedup / benchmark 探测 | 独立 |
| `validate_universe.py` | 校验 universe CSV / PIT manifest schema | 独立 |
| `data_tools/sync_xtquant_to_duckdb.py` | v0.3 同步链路（写项目自有 DuckDB） | 独立，绝不进回测主线 |

## §4 yaml 配置完整 schema

最小有效配置（5 段全部存在）：

```yaml
backtest:
  name:           baseline                # 必填，决定结果目录后缀
  start_date:    "2025-09-02"             # 必填，"YYYY-MM-DD"
  end_date:      "2025-09-26"             # 必填
  initial_cash:  1000000.0                # 必填
  benchmark_code: null                    # 可选，null = 禁用基准
  benchmark_db_path: "F:/backtest_workspace/data/duckdb/benchmark_index.duckdb"  # 可选

data:
  source:     jince_zhisuan               # 必填，jince_zhisuan | qmt_self_owned
  path:       "E:/金策智算/_internal/databases/duckdb/quantifydata.duckdb"
  adjustment: hfq                         # 强制 hfq

universe:
  csv: "backtest/data/universe/strategy_pool_base.csv"   # 二选一
  # pit_manifest: "backtest/data/universe/pit_manifests/p2_1_pit_top100_monthly/index.json"

execution:
  price:           next_open              # next_open | close
  slippage:        0.001
  commission_rate: 0.00025
  tax_rate:        0.0001

strategy:
  max_positions:           5
  rebalance_policy:        daily          # v0.2 仅 "daily"
  min_score:               60.0
  min_core:                32.0
  max_bias5:               10.0
  max_daily_pct:           9.0
  sector_heat_mode:        zero           # v0.2 仅 "zero"
  score_gap_threshold:     15.0
  early_stop_days:         3
  early_stop_loss:         -0.05
  stop_loss:               -0.08
  warning_score_threshold: 50.0
  early_stop_holding_days: 5
  early_stop_min_return:   0.03
```

### 4.1 字段语义速查

| 字段 | 含义 |
|---|---|
| `strategy.max_positions` | 同时持仓上限（顶仓时换仓阈值生效）|
| `strategy.min_score` | 6+2 总分过滤线，低于此分不进 buy_candidates |
| `strategy.min_core` | 6 维核心子和过滤线（不含情绪/板块），防止依赖噪声维度 |
| `strategy.max_bias5` | 5 日 bias 上限（百分比），过滤已涨过头的股 |
| `strategy.max_daily_pct` | 当日涨幅上限，过滤涨停或接近涨停（默认 9% 留 1% 缓冲）|
| `strategy.score_gap_threshold` | 换仓阈值：候选 - 持仓最弱评分 ≥ 阈值才换 |
| `strategy.early_stop_days` / `early_stop_loss` | 短期止损：持有 ≥N 天且收益 < 阈值 → 卖出 |
| `strategy.early_stop_holding_days` / `early_stop_min_return` | 长期淘汰：持有 ≥N 天且收益 < 阈值 → 卖出 |
| `strategy.stop_loss` | 底线清仓（不论持有天数，亏损 ≥ 阈值即卖）|
| `strategy.warning_score_threshold` | 预警卖：持仓评分跌破此线 + 满足其他条件 |

### 4.2 grid yaml schema

```yaml
batch:
  id:   <短标识，决定 batch_summary CSV 文件名后缀>
  base: <基准 yaml 路径，相对仓库根>

grid:
  <dotted-path>: [v1, v2, v3]    # 笛卡尔展开
  strategy.max_positions: [3, 5, 8]
  strategy.min_score: [55.0, 60.0]
  # 留空 grid → 仅跑 base
```

> dotted-path 规则：`a.b.c` 等价于 `cfg["a"]["b"]["c"] = value`。中间层缺失会自动创建 dict。

## §5 数据源说明

### 5.1 两种 source（必须显式声明，无 auto detector）

| data.source | 表 | 日期列 | 去重责任 | 默认过滤 | 适用阶段 |
|---|---|---|---|---|---|
| `jince_zhisuan` | `dat_day` | `trade_time TIMESTAMP WITH TIME ZONE` | reader 强制 `QUALIFY ROW_NUMBER() PARTITION BY (code, CAST(trade_time AS DATE))` | 无 | v0.2 / 短样本验证 |
| `qmt_self_owned` | `dat_day` | `trade_date DATE` | 上游 sync 已保证 `(code, trade_date, adjustment, source)` 唯一 | `adjustment='hfq'` + `source='xtquant'` | v0.3 主路径 |

### 5.2 路径

```text
jince_zhisuan：E:/金策智算/_internal/databases/duckdb/quantifydata.duckdb     (永久只读)
qmt_self_owned：F:/backtest_workspace/data/duckdb/qmt_market_data.duckdb       (sync 写、reader 只读)
benchmark：    F:/backtest_workspace/data/duckdb/benchmark_index.duckdb        (索引数据)
```

### 5.3 reader 输出统一 schema

无论 source 哪种，reader 对外返回的列固定：

```text
date         DATE
open/high/low/close   float64
vol          int64
amount       float64
```

### 5.4 WAL 警告

`jince_zhisuan` 路径下 reader 启动时会检测 `quantifydata.duckdb.wal`：
- 存在 → stdout + logs.txt 打印 `⚠️ ...金策智算可能正在同步...`
- summary.json 落 `data_wal_detected: true` + `data_concurrent_sync_warning: true`
- reader 不中断，但你应该知道 data_hash 此刻不稳定

## §6 universe 两种形态

二选一，**不可同时声明**：

### 6.1 CSV 单池（静态）

```yaml
universe:
  csv: "backtest/data/universe/strategy_pool_base.csv"
```

CSV schema（SPEC §2.5）：

| 列 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `code` | str | ✅ | `^\d{6}\.(SZ|SH)$`；不符的行被忽略并落 logs |

| `sector` | str | ⏸ | 预留，zero 模式下不用 |
| `enabled` | bool | ⏸ | 缺失视为 true；false 行被跳过 |


## §8 strategy_core 接口契约（替换 / 扩展打分逻辑）

### 8.1 入口函数：`evaluate_day`

签名（**冻结，不可改顺序 / 改名 / 改类型**）：

```python
def evaluate_day(
    current_date,        # str, "YYYY-MM-DD"
    market_window,       # dict, code -> DataFrame[date, open, high, low, close, vol, amount]
    positions,           # list[dict]，结构见 03_interface_freeze.md §2
    cash,                # float
    universe,            # list[str]
    account_state,       # dict，§3
    strategy_config,     # dict，§4
    aux_data,            # dict, §5
):
    """返回 StrategyDecision dict（§6）"""
```

调用约束：
1. **纯函数**：相同入参必须返回相同结果（reproducibility 测试依赖）。
2. **不可 import** `xtquant / passorder / get_trade_detail_data / ContextInfo`。
3. **不可 IO**：不读文件、不写文件、不联网、不打印副作用日志（必要日志返回 `decision["logs"]`，由引擎统一持久化）。
4. **Python 3.6 兼容**（hardline）：禁 `dict[str,...]` / `str|None` / walrus / match-case / dataclass / 复杂 f-string。
5. **8 参数顺序**冻结，不调整。
6. `market_window` 切片仅含 `current_date` 及之前的数据（引擎负责切片），strategy_core **严禁**读取未来数据。

### 8.2 返回值：StrategyDecision

```python
{
  "sell_decisions":     [{"code","action","target_volume","reason","layer","priority","diagnostics_ref"}, ...],
  "buy_candidates":     [{"code","score_total","score_core","bias5","daily_pct","rank","target_weight","target_cash"}, ...],
  "target_positions":   [],   # MVP 留空，v0.3+
  "blocked_candidates": [{"code","reason"}, ...],
  "diagnostics": {
    "scores":          {code: {...}},
    "filter_counts":   {"blocked_min_score":..., "blocked_min_core":..., ...},
    "warnings":        [str, ...],
    "trigger_counts":  {"early_stop":..., "stop_loss":..., "replace":..., "warning":..., "confirm":...}
  },
  "logs": [str, ...]
}
```

详见 `03_interface_freeze.md` §6（卖出 reason 枚举见 §7，blocked reason 见 §8）。

### 8.3 想替换打分逻辑？

两条合规路径：

**路径 A：仅替换 6+2 评分（推荐，影响面小）**

修改 `backtest/strategy_core/scoring_adapter.py::score_universe`，但保持：
- 输入：`market_window, sector_heat_mode, aux_data, return_warnings`
- 输出：`(score_records: dict[code, dict], warnings: list[str])` — `score_records[code]` 必须含 `score_total / score_core / bias5 / daily_pct` 等下游 decision 层依赖的字段（具体见 03 §6 diagnostics.scores）。

**路径 B：完全自定义 evaluate_day**

新建 `backtest/strategy_core/<your_strategy>.py`，导出与 `interface.evaluate_day` 同名同签的函数；改 `daily_engine` 的 evaluator 注入点（v0.2 是硬编码，需要 Hermes 评审才能接入新 evaluator）。**未经 Hermes 评审，不得**自行替换 evaluator。

> v0.3 路线图里的「主升浪一体化策略」走的就是路径 B，但需先过 V1 SPEC（参考 `01_cc_engineering_review.md` §九）。

### 8.4 想加一个新维度（如 turnover）？

E0 设计已给出方案（`23_e0_next_scoring_design.md`）：
- **不**作为 6+2 第 9 维（污染评分语义）
- 作为**独立 channel**（hardfilter 或排序权重），在 `decision.py` 的过滤链或排序步骤注入
- 必须留 ablation 开关 + L0–L4 矩阵
- 必须先有 SPEC，CC 再 ODM 实现

## §9 复现指纹

任何一次回测都应该可以**完全复现**。三个 hash 联合签名：

| hash | 输入 | 失效条件 |
|---|---|---|
| `config_hash` | yaml 原文（leaf 展开后的 JSON 序列化） | yaml 任意字段改动 |
| `data_hash` | `db_path + db_mtime + adjustment + requested_start/end + actual_min/max + n_codes + n_rows_after_dedup + dedup_count + universe_hash` | DuckDB 文件被刷新、universe 增删、区间变 |
| `universe_hash` | sorted(universe code list) | universe 行增删 |

> 复现一个老结果：拿到老 summary.json → 找出 `data_path / data_mtime / config_hash / universe_hash` → 还原 yaml + 确认 DuckDB 文件 mtime 一致 → 重跑应得相同 hash。`data_mtime` 不一致就放弃复现，重跑只能拿新数据再跑。

## §12 常见错误与排错

| 现象 | 可能原因 | 处理 |
|---|---|---|
| `FileNotFoundError: DuckDB not found` | DuckDB 路径错（盘符/大小写）| 检查 `data.path`；E 盘是金策智算 SSD（2026-06-14 起从 F 迁过来），不是 F 盘 |
| `data.source must be one of ('jince_zhisuan','qmt_self_owned')` | source 未声明或拼错 | yaml `data.source` 必须显式声明，无 auto detect |
| `yaml universe must set either 'csv' or 'pit_manifest'` | universe 段空或两个字段都没有 | 二选一 |
| `PIT manifest produced empty union` | manifest 所有 snapshot 的 `n_chosen=0` | 检查 manifest 数据是否生成完整 |
| reader 启动打印 ⚠️ WAL 警告 | 金策智算正在同步 | 不阻断，但 data_hash 不稳定；同步结束再重跑 |
| 结果 `is_short_sample=true` | 样本期 < 12 个月 | 决策 H：仅作管线验证，**不可作策略结论** |
| `benchmark_available=false` | DuckDB 中无指数数据或 benchmark_code=null | 决策 E：默认禁用；超额收益 / 信息比率均 null |
| 0 笔成交 | universe 与 DuckDB 数据范围零重合，或所有候选被过滤 | 看 `diagnostics.filter_counts` 哪一项最大 |
| 结果不可复现 | `data_mtime` 已变 | 不能复现；记录新 data_hash 重跑 |
| `clean_results.py` 改慢了主线 | 违反硬约束 #6 | 必须独立 CLI；reader/engine 不得 import |
